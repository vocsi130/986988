Introduction to Servlets Quiz

1. What is the difference between a web server and a web container?
Web Container or Servlet Container or Servlet Engine : is used to manage the components like servlets, JSP. It is a part of the web server. It is web based, runtime environment to support J2EE components. Just as a "servlet container" (such as provided as Tomcat) is a runtime environment for java servlets.
Web Server or HTTP Server: a server which is capable of handling HTTP request send by a client and respond back with a HTTP response. Web server only serves static web pages. Apache, Apache/Tomcat, ISS and JBoss/Tomcat are all examples of web servers.
Having web container implies having a web server. But a web server need not have anything to do with containers.
2. What is a servlet?
A servlet is a Java class that extends the capabilities of servers that host applications access by means of a request-response programming model.
3. How do web servers and web containers interact with servlets?
Web servers interact with servlets with the help of containers. Containers manage the communication between the web servers and servlets. When web servers request for a servlet . Then containers search for servlets based on the URLs. It interacts with servlets with following steps:
  • Container Creates and initialize the servlets.
  • Containers invokes the service method of the servlets based on the method specified GET or POST.
  • Servlets compute the request object and returns dynamic HTML page stuffed in the response object and send it to the containers.
  • Finally containers send the dynamic HTML page to the web servers.
4. Who creates request objects?
Containers create the request objects.
5. What are the steps in the servlet life cycle?
he steps in servlet life cycle are as follows:-
  • Load servlet class (Class c = Class.forname(...))
  • Instantiate servlet (c.newInstance());
  • init() called only once in the servlet's life. Must complete before Container can call service().
  • service() (called for each request, each request runs in a separate thread)
  • destroy() (called only once)
6. Who call init and when?
Container call init when the servlet is initialised.
7. Which of init, service, and doGet should you override?
We need to override doGet method in the servlet class.
8. In what sense are servlets multi-threaded?
Container automatically creates a new java thread for every servlet request instead of running it in different process. Service method of servlet can be called multiples times each call is run in different thread.
9.What are the implications of this for servlet instance variables?
Servlet instance variables are not thread safe as the container does not create a new instance of the Servlet class for each request. It reuses an existing one.
